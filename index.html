java 测试

单元测试框架 - Junit

[Junit] 是 java 单元测试的框架，可以通过 maven
或者 gradle 方式管理。具体使用方法参考官方网站.

断言框架

fest 断言框架

fest 是一种断言框架，而且 IDE 可以显示所有可供使用的 Assertion. 而且
为 java 的反射机制提供了灵活的 API.
fest 可通过 maven 或者 gradle来管理，选择到 mvnRepository 下载依赖。

hamcrest 断言框架

hamcrest 源于 java， 但现在可支持多种语言，如 Phyton、Ruby、Objective-C、PHP、Erlang 和 Swift。
最大的特点是它的匹配器， 可以自由组合创建灵活的表达式， 也可以自定义你所需要的各种匹配器，从而满足
不同测试的需要。

同样的， hamcrest 也可以通过 maven 或者 gradle 进行管理，选择到 mvnRepository
下载依赖。

对我自身而言，在不需要自定义一些 matchers 时， fest 断言框架的语法可读性更高一些，更像 BDD 风格。相比
之下，hamcrest 更像 TDD 风格。

Mock 框架 - mockito

mockito 是 java 单元测试的 mock 框架。你可以使用 mock、stub、spy方法。
而 mockito 能够所有被 mock 记录函数的交互（包括是否被调用、调用几次、使用什么参数调用等等）其功能
如下所示：

  1. 使用 mockito 验证行为。
  2. 使方法调用返回特定期望的值，也称 stubbing.(默认情况下， 所有有返回值但是没有被 stub 的方法都会返回默认值， 如：
    int/Integer 类型 会返回0， boolean/ Boolean 类型返回false， 其他类型则会返回 null)。
  3. 参数匹配
  4. 验证方法是否被调用，或者被调用的特定的次数。
  5. 调用无返回值的方法时， 返回异常。（使用 doThrow）
  6. 验证函数被调用顺序
    更多功能参考 mockito 官网.

如果使用 maven 或者 gradle 进行管理时， 可选择到mvnRepository
下载 mockito 库。

AngularJs-前台框架

定义

AngularJs 是一套由Google打造的JavaScript前端框架，对于动态网页的开发尤其适合。
做过 JS 开发的人都会知道，对于用 DOM 或者 JQuery 动态操纵view上的元素是
一件比较麻烦的事情。但是当使用了 AngularJs 之后，这个问题便迎刃而解。

特性

  1. MVC
    MVC 是模型(model)－视图(view)－控制器(controller)的缩写，使用一种业务逻辑、
    数据、界面显示分离的方法来组织代码。其中：
    Model ：就是应用中的数据，一般式 JavaScript 对象。
    View ：是AngularJs 解析后渲染和数据绑定后生成的HTML。
    Controller : 可以初始化初始化页面对象，并且定义和实现一些 View 上需要的方法。

  2. 数据双向绑定
    通过 $scope 是 AngularJS 中的一个 JavaScript 对象， 可以完成数据绑定的任务。
    在 Controller 中使用 $scope 可以针对特定的 View 来扩展数据和特定的功能。无论是
    页面上被绑定的元素还是 controller 中的 $scope 的数据值发生了变化，另一方的值也会
    发生变化。

使用方法

  1. 建立index.html文件
    在 body 标签上其中使用 ng-app ，给出 app 的名称，然后在 body
    内的 div 上使用 ng-view=””, 最后引入所有需要的 js 文件（包括一些库函数和controller文件）。

  2. 建立app.js
    使用 var app = angular.module('myApp', [ 'ngResource','ngRoute','checklist-model']);，声明 module 的名称（此处声明为MyApp）。然后使用.config 和 .when来配置模板的 url 和 controller。

更多详情：请参考AngularJs官网
