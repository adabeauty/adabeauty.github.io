JS后台debug方法

一直以来，我对于后台的调试方法都只是简单的在觉得可能出现错误的地方使用console.log( )，在控制台打印信息，
直到最近接触到了Node.js里一个插件inspector。这是一个基于 Node.js的调试器，通过它，你可以像调试前端代码一样来调试你的后台代码。

首先，全局安装inspector

$ npm install -g node-inspector

启动inspector

进入到需要调试的项目目录

运行该项目(app.js为你后台程序的入口文件)

在浏览器打开  localhost:8080/debug  端口

此时在该页面左侧会出现source字样，点击后可看到该项目源代码

设置断点

在需要调试的地方设置好断点

运行需要调试的项目

在3000端口运行需要该项目（本人的项目是在3000端口下运行）

查看调试结果

在8080端口，查看调试结果
