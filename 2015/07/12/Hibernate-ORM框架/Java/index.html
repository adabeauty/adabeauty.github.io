Hibernate 介绍

Hibernate是一个开源的ORM（object relations mapping）框架，
用面向对象的思想来操作数据库，节省了我们开发处理数据的时间。它对JDBC进行了非常轻量级的对象封装，
可以应用在任何使用JDBC的场合。

Hibernate 配置

Hibernate 的配置主要包括两方面的配置。

对需要映射的对象进行配置

第一个是要对需要映射的对象进行配置。可以采用xml文件的方式或者
直接使用注解Annotation的方式进行配置。

同理，可以配置其他的属性，如name，gender，email和age，分别来对应到USER表中的属性。完成对映射的配置。

使用注解方式配置的时候并不需要增添多余的配置的文件，只需要在需要被映射的实体上添加注解即可。具体配置如下：

该方式通过@Entity注解来表明该类是一个要映射到数据库的实体，@Table指明了与该实体对应的数据库中
的表。通过@Id指明主键，当然你也可以添加‘@Generator’注解来指明主键生成方式。其他的属性通过@Column
来指定与数据库对应的列。

必须指明的是，在创建这个Entity的时候必须含有getter和setter方法以及午餐构造函数，因为Hibernate在执行
映射的时候会自动去寻找对应属性的getter和setter方法。

以上这两种方式都可以完成对映射对象的配置。第一种方式看似复杂一点，需要书写一个配置文件，但其可以在数据表名或者
属性名称发生变化时仅仅修改这一个配置文件，便可使得程序继续正常工作，而不用去修改实现代码。

对于方式二，我个人喜欢的原因是它比xml文件的可读性要高，让人可以一眼看出来每个注解的作用。当然缺点就是如果表发生
变化的时候，需要重新返回到Entity的实现代码部分进行修改，当有多个Entity需要配置的时候，这种方式便显得优势不再突出。

连接数据库部分的配置

上一部分的配置仅仅是把映射对象配置完成了，但是此时，Hibernate其实还并不知道要连接哪个数据库，怎么连接。所以接下来
我们将会对数据库连接这一部分进行配置。

在<session-factory>中配置关于连接的这部分的操作。

首先，配置hibernate.connection.url
设置数据库连接的url，username和password来设置连接数据库时候使用的用户名和密码。
pool_size设置连接池中的连接数量，show_sql设置为true后，Hibernate可以在执行完SQL
语句后显示执行的SQL语句内容，dialect指定的时候hibernate使用的方言，这个跟使用的数据
库有关，我这里使用的时MySQL的数据库，所以选用org.hibernate.dialect.MySQLDialect.

Hibernate 工作原理

Hibernate使用部分，我是自己书写了一个Util来帮助我产生session.

具体的工作原理如下：

  1. Hibernate 初始化，读取Hibernate 的配置信息（即Hibernate config.xm文件），一个Configeration 实例代表Hibernate 所有Java类到Sql数据库映射的集合.

    Configuration configuration = new Configuration().configure();

  2. 创建一个Session Factory，把Configeration 对象中的所有配置信息拷贝到
    SessionFactory的缓存中。 SessionFactory的实例代表一个数据库存储源，创建后
    不再与Configeration 对象关联。

  3. 调用SessionFactory创建Session的方法。

  4. 通过Session 接口提供的各种方法来操纵数据库访问。比如增加一条数据可以用session.save(Object object)，
    删除一条数据可以 session.delete(Object object)，更新数据session.update(Object object),
    查询可以使用Query query = session.createQuery(hql)的方法。

    Hibernate会把session缓存中的所有SQL语句一起执行，对于更新、删除操作也是采用类似的机制。
