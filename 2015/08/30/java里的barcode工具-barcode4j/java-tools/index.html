由于项目需要，近两天看了一些关于生成条码的插件。最开始我是使用了一个叫做niceLable的软件。这个软件可以安装在Windows平台上，有图形化界面，用户可以根据自己的需要来设计需要的条码。也可以通过导入文件作为database，来批量生成条码。

然而，如果我们想在我们的java web程序中来产生条码的时候，我们则不能再依靠第三方的软件。幸好现在有多种java 插件可以用来生成条码，今天我就给大家介绍一下我了解到的两种barcode4j和 zxing。当然这两种工具都可以支持生成一维码和二维码，但是呢今天只跟大家先介绍一下一维码的生成，因为二维码的人家也没有写过啦~ 不过我相信原理都是类似的。

今天呢，我就先奉上barcode4j实现条形码生成的方法。

barcode4j是由java语言编写的一种条码生成器，采用Apache License V2.0许可。可以用来生成一维码和二维码，而且可以输出SVG、EPS、Bitmaps(如png、jpg)和java2D类型输出。而且text类型的输出类型目前正在测试之中。

然而要想使用barcode4j，首先必须先下载它依赖的两个包，分别是：barcode4j和avalon.framework

可以支持code39、code128、EAN-128、GS1-128(基于code128的一种编码)、codabar、UPC-A和UPC-E等多种
编码方式。

来说了这么多我们先看一下使用barcode4j后产生的效果吧（我这里使用的是code39编码，输出是选择的EPS格式，当然你可以选择你需要的编码类型）

创建一个CanvasProvider(根据输出文件不同而异)

CanvasProvider函数有两个参数，第一个参数用于确定是否使用output的命名空间，false代表不使用output的命名空间。对于第二个参数它只可以是0，90，-90，180，-180，270和-27，一般数情况下我们都直接指定它为0. 可以执行绘制条码的指令，然后将他们转换成需要输出的模式。所以选择不同类型的输出文件，这一步的操作方法也不一样，那么我们一个个来说举例说明。

不知道有木有小伙伴跟我一样并不知道SVG格式是什么格式，我也是写这篇博客的时候才去查了一下子。SVG（Scalable Vector Graphics）也是一种图像文件格式，是一种用XML定义的语言，用来描述二维矢量及矢量/栅格图形。

EPS 类型

其实我在我的程序里采用的是这种EPS格式。关于EPS格式是什么类型的格式，这个我也并不是很了解，附上一段我在网上看到的解释吧：

EPS文件是目前桌面印刷系统普遍使用的通用交换格式当中的一种综合格式。EPS文件格式又被称为带有预视图象的PS格式，它是由一个PostScript语言的文本文件和一个（可选）低分辨率的由PICT或TIFF格式描述的代表像组成。 EPS文件就是包括文件头信息的PostScript 文件，利用文件头信息可使其他应用程序将此文件嵌入文档。

其实我也并不是很理解，但是总之一点，这个格式好多打印机都只支持，所以我果断选择了它。

这里需要说明的一点是使用这种方式必须在生成barcode后调用finish方法。然后你就可以看到项目根目录下多了一个叫做eps_barcode.eps的文件。

Bitmaps 类型

Bitmaps就是位图文件，扩展名为.bmp或者.dib，是Windows上的标准图形文件，它将图像定义为由点（像素）组成，每个点可以由多种色彩表示。Bitmaps类型图像效果非常好，但是是非压缩的，所以需要较大内存。

Bitmaps 类型和EPS类型都是将输出流写到对应的文件之中。

当然你也可以选择生成.jpg格式，但是不推荐，因为.jpg格式是一种有损压缩格式，对比就会受到影响,因此会降低生成的条形码的可读性。

Java2D (AWT) 类型

Java2D是一套用于高级图形和图像处理的类，而AWT(Abstract Window Toolkit)是java抽象窗口工具包，提供了许多设计GUI的组件。
