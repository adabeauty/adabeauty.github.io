flyway - 数据库版本控制工具


什么是数据迁移？什么情况下需要进行数据迁移？

数据库迁移包括两种情况：情况一是数据库迁移，由于项目需要，当前数据库需要从一种类型
迁移为另一种类型，例如从 MySQl 迁移到 Oracle，另一种情况则是很多时候数据需要迁移到一台
新的服务器上，此时也会用到数据迁移

Flyway 特点

Flyway 支持 sql 脚本，支持 Java 代码直接进行操作。
是独立于数据库的应用，是能够管理并跟踪数据库变更的数据库版本控制工具。

而且Flyway 支持包括 Oracle、SQL Server、MySQL、DB2等多种关系型数据库类型。

Flyway 工作原理

对于一个空的数据库，会有一张叫做 SCHEMA_VERSION 的表，这张表可以跟踪数据库的状态。
数据库的迁移会根据版本号进行排序，然后按照顺序应用，每次数据迁移进行后，SCHEMA_VERSION
表都会相应的进行更新。

查看 SCHEMA_VERSION 表时，你可以发现其中有 version_rank、installed_rank、version
以及 description 等信息。
