定义

AngularJs 是一套由Google打造的JavaScript前端框架，对于动态网页的开发尤其适合。
做过 JS 开发的人都会知道，对于用 DOM 或者 JQuery 动态操纵view上的元素是
一件比较麻烦的事情。但是当使用了 AngularJs 之后，这个问题便迎刃而解。

AngularJs是一个扩展性很好的工具集，也可以和多种其他库协同合作。AngularJS的开发团队将其描述为一种
构建动态Web应用的结构化框架。

基本概念

MVC

MVC 是模型(model)－视图(view)－控制器(controller)的缩写，使用一种业务逻辑、数据、界面显示分离的方法来组织代码。其中：
Model：就是应用中的数据，一般式 JavaScript 对象.
View：是AngularJs 解析后渲染和数据绑定后生成的HTML。
Controller: 通过给定我们数据模型的语境，控制器允许我们建立模型和视图之间的数据绑定.
可以初始化初始化页面对象，并且定义和实现一些 View 上需要的方法。
这种表现分离能将应用中的对象很好地隔离开来,因此视图不需要知道如何保 存对象,只要知道如何显示它即可。这也意味着数据模型不需要同视图进行交 互,只需要包含数据和操作视图的方法。控制器用来存放将二者绑定在一起的业务逻辑。

作用域

作用域包含了渲染视图时所需的功能和数据,它是所有视图的唯一源头。可以将作用域理解成视图模型(view model)。
$scope对象在AngularJS中充当数据模型,但与传统的数据模型不一样,$scope并不负责处 理和操作数据,它只是视图和HTML之间的桥梁,它是视图和控制器之间的胶水。

$scope对象的生命周期处理有四个不同阶段，分别是创建、链接、更新和销毁。在控制器或指令创立之初，
AngularJS会用$injector创建一个新的作用域,并在这个新建的控制器或指令运行时将作用域传递进去。

数据双向绑定

通过 $scope 是 AngularJS 中的一个 JavaScript 对象， 可以完成数据绑定的任务。在 Controller 中使用 $scope 可以针对特定的 View 来扩展数据和特定的功能。无论是页面上被绑定的元素还是 controller 中的 $scope 的数据值发生了变化，另一方的值也会发生变化。

使用方法

建立index.html文件

在 body 标签上其中使用 ng-app ，给出 app 的名称（这表明之后的代码全是使用了AngularJs框架的），然后在 body
内的 div 上使用 ng-view=""（此处缺省的部分会通过$routeProvider来根据url动态的加载相应的html文件，
此部分的配置在app.js之中）, 最后引入所有需要的 js 文件（包括一些库函数和所有的controller文件）。

按照这个方式配置而成的index.html就是AngularJs的Template，所有的其他html文件都将会作为partial加载在ng-view=''
的地方，然后渲染成为一个完整的页面。

建立app.js

使用 var app = angular.module('myApp', [ 'ngResource','ngRoute','checklist-model']);，声明 module 的名称（此处声明为MyApp）。然后使用.config 和 .when来配置模板的 url 和 controller。

在AngularJS中,模块是定义应用的最主要方式。模块包含了主要的应用代码。一个应用可 以包含多个模块,每一个模块都包含了定义具体功能的代码。

AngularJS允许我们使用angular.module()方法来声明模块,这个方法能够接受两个参数, 第一个是模块的名称（此处为users_management）,第二个是依赖列表,也就是可以被注入到模块中的对象
列表（这里仅仅添加了路由的依赖）。

按照如上配置后，$route会一直观察$location.url()的变化，然后为其匹配相应的template和controller。当url的路径为/users的时候，views/users.html作为template的url，对用的controller为usersController。
