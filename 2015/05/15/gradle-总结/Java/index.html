优点

  1. 同时提供了构建和依赖管理的能力
  2. 相比于类似Maven的一些构建工具使用xml进行配置书写，gradle使用Groovy作为构建语言，
    大大提升了自动化过程的可编程能力和脚本的可读性，插件的编写、构建逻辑的复用也变得更为容易。
  3. 完全支持Maven的库（repository）

安装配置

一种是从官方下载压缩包然后解压配置环境变量。 另一种是可以直接通过命令行安装执行。

基本概念

首先，我们需要了解gradle默认在src/main/java下寻找源代码，在src/test/java下寻找测试代码，
src/main/resource下找到资源（resource 下的文件会被包含在jar包中，而且文件下的文件会被加
到classPath中执行）。当然我们也可以通过自己设定的方式改变源代码的位置。在使用IDE进行开发时
候，可进入File下的ProjectStructure指定该项目的srcs、tests和resources的位置。

其实，在gradle中有两个名词，一个是project，另一个是task。Project代表着整个项目，而一个project
可以由一个或者多个task组成。当然，你也可以通过如下命令来查看所有的tasks.

执行完后会发现，可以由Application tasks、Build tasks、Build Setup tasks、Documentation tasks、Flyway tasks等。总而言之，使用 gradle + task 名称即可执行相应的task.

仓库管理

仓库是一种存放依赖的容器，每一个项目都可以有一个或者多个仓库。
gradle支持一下三种仓库格式：

  1. Ivy仓库
  2. Maven仓库
  3. Flat directory仓库

这每一种仓库类型都有自己的配置方式,需要在build.gradle文件中进行配置。

依赖管理

当想要声明一个新的依赖时，我们需要两步：

  1. 指定依赖的配置（如指定compile、testCompile等）
  2. 声明所需要的依赖

依赖配置项

可通过使用配置项来声明项目的外部依赖，依赖配置项有很多，具体如下：

  1. compile: 声明当项目被编译时需要的依赖
  2. runtime: 声明项目在运行时所需要的依赖
  3. testRuntime: 代码是在运行测试代码的时候是必须的
  4. archives: 包含项目生成的文件（如Jar文件）
  5. default: 包含运行时必须的依赖

声明项目依赖

项目中最普遍的依赖外部依赖，这些依赖存放在外部仓库中，一个外部依赖可以由以下属性
指定：

  1. group: 指定依赖的分组（Maven中就是groupId）
  2. name: 指定依赖的名称（Maven中就是artifactId）
  3. version: 就是指定外部依赖的版本（Maven中就是version）
