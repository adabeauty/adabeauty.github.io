Spring 框架

Spring是一个轻量级的控制反转(IoC)和面向切面(AOP)的容器框架。
从简单性、可测试性和解耦合的角度而言，任何Java应用都可以从Spring中受益。

组成

首先，给大家贴上一张都很熟悉的图：



从这个图里可以看出来，Spring 是一种分层架构，由七个定义良好的模块组成。
而组成 spring 的每个模块可以单独存在，也可以与其他一个或者多个模块联合实现。
每个模块的功能如下：

  1. Spring Core：主要组件式 BeanFactory，使用控制反转模式（IoC）将配置和依赖性规范和实际应用
    分开。
  2. Spring context：是一个配置文件，向 Spring 框架提供上下文信息
  3. Spring DAO： JDBC DAO 抽象层提供了有意义的一场节凑层次，用来管理和处理不同数据库供应商
    提供的错误信息。在通常情况下，在DAO层与数据库进行交互。
  4. Spring ORM: 支持多种ORM框架，提供了ORM的对象关系工具。包括JDO、HibÂernate 和
    iBatis SQL Map（其实我只知道 Hibernate ~(≧▽≦)/~啦啦啦）
  5. Spring AOP: 将面向方面的编程集成到 Spring 框架中。
  6. Spring Web: 为基于 Web 的应用程序提供了上下文。
  7. Spring MVC: MVC 框架是一个全功能的构建 Web 程序的 MVC 的实现。视图支持JSP、Velocity、Tiles、iText 和 POI。（这个嘛，我只用过 JSP 而已啦）

特点

轻量级

Spring 在大小和开销方面而言都是轻量的。完整的Spring框架可以在一个大小只有
1MB多的JAR文件里发布。并且Spring所需的处理开销也是微不足道的。此外，Spring是非侵入式的：
典型地，Spring应用中的对象不依赖于Spring的特定类。

控制反转(IoC)

被动的方式传进来，而不是在这个对象的内部自己创建。在对象初始化时，容器就会主动请求将
