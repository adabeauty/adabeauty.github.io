概述

jQuery选择器按照功能主要分为”选择”和”过滤”，并且可以配合使用。主要的区别是”过滤”的作用是指定条件从前面的匹配的内容中筛选，而“ 过滤”选择器也可以单独使用, *表示从全部中筛选. 比如:$(':[title]')等同于$('*:[title]')。而”选择”功能的选择器则不会有默认的范围, 因为作用是”选择”而不是”过滤”。

jQuery的选择器可以使得我们使用Id、TagName等多种方式获取到我们想要的得到的DOM对象。而在jQuery中，使用$来表示对jQuery的引用。

分类

jQuery的种类繁多，包括：jQuery元素选择器、jQuery属性选择器以及各种选择器方式等。

Hierarchy 层级选择器

  1. ancestor descendant: 选择满足ancestor下的descendant

  2. parent > child: 选择parent 的直接子节点child

  3. pre + next: pre和next是同级别元素，选中跟在pre后的next元素

  4. pre ~ siblings: 选择pre后面的根据siblings过滤的元素

Basic Filters 基本过滤器

  1. :first
    匹配找到的第 一个元素

  2. :last
    匹配找到的最后一个元素

  3. :not(selector): 去除所有与给定选择器匹配的元素

  4. :even或者:odd : 匹配所有索引值为偶数或者奇数的元素，从 0 开始计数

内容过滤器 Content Filters

  1. :contains(text):
    匹配包含给定文本的元素

  2. :empty
    匹配所有不包含子元素或者文本的空元素

  3. :has（selector)

  4. :parent
    匹配含有子元素或者文本的元素

可见性过滤器

  1. :hidden
    匹配所有不可见元素

  2. :visible
    匹配所有的可见元素

属性过滤器

  1. [attribute]
    匹配包含包含给定属性的元素

  2. [attribute=value]
    匹配给定的属性是某个特定值的元素

  3. [attribute!=value]
    匹配给定的属性是不是某个特定值的元素

  4. [attribute^=value]
    匹配给定的属性是以某些值开始的元素

  5. [attribute$=value]
    匹配给定的属性是以某些值结尾的元素

  6. [attribute*=value]
    匹 配给定的属性是以包含某些值的元素

子元素过滤器 Child Filters

  1. :nth-child(index/even/odd/equation)
    匹配其父元素下的第N个子或奇偶元素

‘’:eq(index)’ 只匹配一个元素，而这个将为每一个父元素匹配子元素。:nth-child从1开始的，而:eq()是从0算起的！

  1. :only-child
    如果某个元素是父元素中唯一的子元素，那将会被匹配如 果父元素中含有其他元素，那将不会被匹配。
