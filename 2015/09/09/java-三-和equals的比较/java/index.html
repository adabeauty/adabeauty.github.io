说到==和equals我必须吐槽一下我曾经在不懂得他们区别的时候遇到的各种坑，我以为我用的==或者equals方法可以完成我想要的判断，信誓旦旦的就执行程序去了，结果……然后也看不出大问题，就一步步调试，才发现他所谓的==和euqals根本没起到我想它起的作用。

说实话物品确实是个比较懒的人，这个问题困扰我好多次了，直到最近我才真正的去了解他们之间的区别于联系。其实很简单，那么我们就一起来看看吧，相信你们以后也不会再出现我那样的问题……

要想讲清楚这个问题，我们还必须回想一下上一篇中讲过的java数据类型：primitive类型和引用数据类型。这两种类型在讲解==和equals的区别的时候起到了巨大的作用。

复习一下，primitive基本类型保存的是变量的真实值（包括八种：byte,short,char,int,long,float,double,boolean），而引用类型则是保存变量的引用，即它们在内存中被分配的地址。好的，知道这些，我们就可以正式开始学习 == 和 equals的区别了。

引用类型

引用类型在==和equals使用上是有点差别的。

== 使用

对于引用类型而言，==比较的是两者在内存中存放的地址，除非是一个new出来的对象，否则即使数据完全相同的两个对象使用==比较，结果也是false。

equals 使用

在java中所有的类都是继承于Object这个类的，在Object中就定义了一个equals方法，这个方法是用来比较对象的内存地址的，也就是说和==功能相同。但是一些库类当中这个方法被覆盖掉了，比如String、Integer、Date等类中，equals就有其自身的实现，而不是去比较内存地址了。这一点大家可以通过查看这些库类的equals方法的源码来了解它们的实现方式。
