今天主要给大家介绍一下Java中的基本概念。

JDK、JRE和JVM

JRE

JRE（Java Runtime Environment）Java运行环境，也就是Java平台。所有的Java程式都必须在JRE上才能运行。
JRE是Sun公司发布的一个更大的系统，它里面就有一个JVM，我们从Sun下载JRE的时候就可以发现它里面有各种各样不同的版本，因为JRE跟具体的操作系统和CPU的结构有关系。任何写好的程序都需要操作系统帮我们运行，所以我们的java程序也需要JRE提供运行的平台。

JDK

JDK (Java Developer Kit)是Java开发工具包，相当于Java的库函数，是编译、运行java程式的工具包。
当我们进行java程序开发的时候，我们必须下载JDK，如果你打开JDK的文件夹你会发现




可以看到这里有一个JRE文件夹，这个JRE是JDK的java代码运行的环境。为了保证JDK的完整性和独立性，在JDK的安装过程中，也安装了它自己的JRE.

JVM

JVM（Java Virtual Machine）Java虚拟机，是JRE的一部分，它就是虚拟出来的一个专门运行java编译后文件的地方。对于java来说，最大的特点就是跨平台性，而这部分的实现就主要是通过JVM来实现的，因为JVM会将编译后的字节码转换成平台能够理解的形式来运行，从而实现了跨平台。

编译器和解释器

编译器

编译器完成编译的功能，即将我们编写的.java文件编译成一种中间代码-字节码（一种可以被java解释器解释的独立于平台的代码），也就是.class文件。

在这里可能一些初学者会纳闷了，要运行java编译程序还需要编译？我怎么什么都没做，就在IDE点了一下运行按钮，程序就执行了。这是因为像eclipse，InteliJ这样的开发环境内置了编译器，它帮我们完成了编译这一部分功能，所以我们才不用显式的自己使用编译器来编译。

高级学习：
大家应该都知道Java是一种强类型语言，这代表着它不能允许变量保存类型的数据。这是关键的类型安全性功能。所以类型错误会在编译时候就被发现，然后执行的时候数据就是类型安全的，就不需要再保存数据的类型。即在编译的时候会对数据类型进行擦除。

解释器

通过解释器，每条java字节都会被分析，然后再计算器上运行，所以只需编译一次。而java程序是运行时解释执行的。

Java的工作方式

工作流程

首先，我们作为一名程序猿，会先编写自己的源代码（通俗讲也就是我们的.java文件啦）。
然后该编译器发挥作用了。它在执行编译功能，再通过编译后，会将我们的源码.java编译成.class文件，这个文件里都是由字节码（这是我们的解释器可以看懂的一种语言）组成的。
最后就该我们的JVM上场了。启动JVM来运行.class文件，然后解释器会将这些字节码转化成平台能够理解的形式，然后再由JVM来运行。
